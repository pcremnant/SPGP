# 스마트폰 게임 프로그래밍 3차 발표
## 게임 컨셉
### 2D 횡스크롤 디펜스 게임
### 일자로 된 맵 양 끝에 플레이어와 적군 성 배치
### 성에서 자원을 활용해 유닛 생산, 자동으로 적 성을 향해 공격
### 적 성을 부수거나 플레이어의 성이 부서지면 게임 종료
### ex > 카툰워즈, 팔라독

## 개발 목표 변동
|계획|실제 진행|
|:---:|:---:|
|[유닛] 아군 4종 / 적군 4종|[유닛] 아군 2종 / 적군 2종|
|스테이지 3종|스테이지 1종|
|게임 로비 및 UI|미구현|
### 일정 배분 실패로 인해 목표 축소

## 사용된 기술
### [수업시간 차용]
### 충돌 박스 설정
### 오브젝트 레이어
### [직접 구현]
### 애니메이션 관리 클래스
### 유닛 오브젝트 상태 관리

## 아쉬운 점
### 일정 배분에 실패하여 사실상 프로젝트를 완성하지 못함
### 안드로이드 스튜디오의 다양한 기능들 (xml을 이용한 레이아웃 구성 등)을 활용하지 못함

## 보충해야 할 점
### 아군, 적군 유닛의 종류 다양화
### 스테이지 갯수 증가
### 스테이지 진행 시 성장요소 추가 (유닛, 성 업그레이드 등)
